Сетевые адаптеры (хост-адаптеры) являются финальным звеном цепочки обмена данными с клиентом. Именно при помощи этого устройства система хранения данных соединяется с «внешним миром»: серверами, рабочими станциями и другими компонентами сетевой инфраструктуры. Сетевой адаптер представляет собой плату с портами интерфейсов (Ethernet, FC, IB, SAS), которая использует разъем на материнской плате или впаивается в нее напрямую

![](../../../images/Сетевые%20адаптеры-05.02.2026-01_02.png)
- типовое исполнение (PCI-E плата расширений, OCP 3.0, mezzanine card);
- шину подключения (PCI-E, PCI 32 bit, PCI-X);
- скорость обмена данными (10Gb/s, 25Gb/s, 100Gb/s);
- поддерживаемые интерфейсы и их количество (SFP+, SFP28, QSFP28);
- протокол передачи данных (Ethernet, Infiniband, Fiber Channel).

Применение сетевой карты позволяет превратить компьютер из изолированной системы в часть комплексной инфраструктуры, с подключением к локальной сети или глобальной. Адаптеры работают по одному из протоколов Ethernet, Infiniband или FiberChannel. По типу передаваемого трафика и назначению подразделяются на:


- NIC (Network Interface Controller) для Ethernet;
- HBA (Host Bus Adapter) для SAN и/или DAS сетей на проколе FiberChannel;
- HCA(Host Channel Adapter) для работы с Infiniband в частности для кластеризирования.


Также сетевые адаптеры классифицируются по типам портов, которые могут быть не только RJ-45, применяемым для подключения к сети посредством витой пары, но и иными:

| Наименование интерфейса | Скорость передачи данных                                                                                                                  |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| RJ45                    | До 25 гигабит в секунду. Для передачи данных используется 4-8 контактная витая пара различных категорий(Cat 8 для 25GBASE-T)              |
| SFP                     | До 1,25 гигабит в секунду. Является стандартном оптоволоконной передачи данных. Заменен стандартом SFP+.                                  |
| XFP                     | До 10 гигабит в секунду. Имеет нестандартные габариты модуля. Способ передачи данных - оптика.                                            |
| SFP+                    | До 10 гигабит в секунду. Самый популярный стандарт подключения в ЦОД.                                                                     |
| SFP28                   | До 25 гигабит в секунду. Является идейным продолжением SFP+. Имеет обратную совместимость с SFP+.                                         |
| QSFP+                   | До 40 гигабит в секунду. Состоит из четырех каналов передачи данных по 10 гигабит.                                                        |
| QSFP28                  | До 100 гигабит в секунду. Состоит из четырех каналов передачи данных по 25 гигабит.                                                       |
| QSFP56                  | До 200 гигабит в секунду. Состоит из четырех каналов по 50 гигабит. Имеет обратную совместимость с QSFP+ и QSFP28.                        |
| OSFP                    | До 400 гигабит в секунду. Имеет внутри себя 8 каналов по 50 гигабит в секунду. Распаивается на самых современных сетевых картах Mellanox. |
| QSFP-DD, QSFP-DD800     | До 800 гигабит в секунду. Самый современный форм фактор коннектора. Обычно используется в коммутаторах.                                   |

Использование нескольких портов на сетевой карте:

|Цель|Как это работает?|Типичный сценарий|
|---|---|---|
|**Увеличение пропускной способности**|Несколько портов объединяются в логическую группу (агрегация, teaming, bonding). Это позволяет суммировать их скорость[](https://www.ittelo.ru/news/setevye-karty-v-serverakh-kak-vybrat-optimalnoe-reshenie/)[](https://forums.tomshardware.com/threads/purpose-of-dual-ports-on-nics.2265343/).|Сервер с двумя портами по 1 Гбит/с может передавать данные со скоростью до 2 Гбит/с[](https://www.ittelo.ru/news/setevye-karty-v-serverakh-kak-vybrat-optimalnoe-reshenie/).|
|**Обеспечение отказоустойчивости**|Настройка режима Active Backup. Если основной порт или кабель выходит из строя, трафик мгновенно переключается на резервный[](https://ya.ru/neurum/c/tehnologii/q/kakie_preimuschestva_daet_otkrytie_dopolnitelnyh_428dec12)[](https://forums.tomshardware.com/threads/purpose-of-dual-ports-on-nics.2265343/).|Критически важный сервер (например, баз данных) должен оставаться доступным даже при проблемах с сетевым подключением.|
|**Разделение типов трафика**|Разные порты выделяются под разные задачи: доступ к данным, служебный трафик, резервное копирование, миграция виртуальных машин[](https://www.ittelo.ru/news/setevye-karty-v-serverakh-kak-vybrat-optimalnoe-reshenie/).|На сервере виртуализации один порт может работать с хранилищем данных (SAN), а другой — обслуживать пользовательские запросы.|
|**Подключение к изолированным сетям**|Каждый порт подключается к физически отдельной сети для безопасности или соблюдения правил[](https://forums.tomshardware.com/threads/purpose-of-dual-ports-on-nics.2265343/).|Подключение одной картой к корпоративной сети с выходом в интернет, а другой — к закрытой внутренней сети (например, для систем видеонаблюдения или управления оборудованием)|
